Верстка

Инфо

  • Кодировка UTF-8

  • Верстка адаптивная, градации по ширине лучше подскажет Дима, скорее всего 2 - мобильники вертикальной ориентации и планшеты горизонтальные++

  • В проекте используется шаблонизатор SMARTY, то есть вся верстка будет раскидана по шаблонам, и страницы будут собираться из них.

  • В качестве JavaScript фреймворка используется JQuery

  • Верстка должна содержать все возможные варианты развития событий - например вывод всплывающих окошек, всех ошибок в инпутах не только внутренней верификацией, но и своими скриптами с произвольным текстом, например после ajax запроса на сервер.



  • По FTP доступна директория для шаблонов (.tpl) и других файлов (директории img, js, css). Это developer (test) версия.
    Директория sandbox - для экспериментов на чистом html. Она доступна по ссылке https://hidesend.com/sandbox/

  • Переключение между test и production версиями по ссылкам http://hidesend.com/goto-test и http://hidesend.com/goto-prod

  • В шаблонах пути к директории изображений, js и css используются с переменной {$VER},
    например: /css/{$VER}/style.css, чтоб было удобно копировать файлы не меняя пути между prod и test версиями сайта

  • Других ссылок на корневые директории сайта нет, то есть если нужна папка например hidesend.com/fonts/ - пиши, я сделаю для prod и test версии.
    Не стоит пихать все в кучу, например ширфты в /css/

  • Основной шаблон - mаin.tpl . В нем основной каркас от <html> до </html>. Внутрь вставляются переменные, содержащие контент в зависимости от страницы

  • Переменные - конструкции вида {$NEWS}, {$BANNER}, итд.
    Они сожержат текст и кусочки верстки, находящиеся в других шаблонах, в итоге составляется страница целиком

  • Так же могут быть условия, означающие будет ли выведено что-то в страницу или нет. Например {if $VER=='test'}Номер ячейки:{$VAULT_NUMBER}{/if}

  • Все условия я пропишу, а так же подскажу что в каких переменных и какие части верстки в каких шаблонах находится
